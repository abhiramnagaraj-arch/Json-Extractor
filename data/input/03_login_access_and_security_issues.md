## How to Resolve Login Failed [Error:V] message seen in Webmail, Due to IdProtect Security Feature

ID: `how_to_resolve_login_failed_error_v_message_seen_in_webmail_due_to_idprotect_security_feat`
Category: Admin Panel/Webmail
Subcategory: General
Source section: How to Resolve Login Failed [Error:V] message seen in Webmail, Due to IdProtect Security Feature.

Alternate questions:
- How do I resolve login failed [error:v] message seen, due to idprotect security feature?
- How can I resolve login failed [error:v] message seen, due to idprotect security feature?
- Login Failed [Error:V] message seen in Webmail, Due to IdProtect Security Feature
- Why am I seeing Login Failed [Error:V] message seen in Webmail, Due to IdProtect Security Feature?

Answer:
Purpose: To explain the cause of the error "Login Failed [Error:V]" and guide both users and administrators on how to verify and approve login patterns under the IdProtect feature. - Background The error "Login Failed [Error:V]" occurs when: IdProtect (network login protection) is enabled for the domain. The user tries to log in from a new or unrecognized network/IP. The login is blocked until the new pattern is verified. This security feature helps prevent unauthorized access and notifies both the user and the admin of any new login patterns. Verification Methods: There are two ways to verify a login pattern: - ✅ 1. User-Based Verification Steps: The user should check their alternate email address or registered mobile number. Open the message received with the verification link and code. Click the link and enter the code provided in the message. Once verified, login will be allowed from that network. - ✅ 2. Admin-Based Verification Steps: Admin logs in to the Admin Email Account. - Navigate to: Admin Page → Other Services → Suspicious Login Report Click on the “Notification Sent” tab (not the default tab). Search for the affected user/email address and click Submit. Review the login pattern details. Click “Accept” to approve the new login pattern.

Keywords: resolve, failed, seen, idprotect, security, general

## How to Resolve Login Failed [Error B]: Not Authorized Due to IdProtect Security

ID: `how_to_resolve_login_failed_error_b_not_authorized_due_to_idprotect_security`
Category: Admin Panel/Webmail
Subcategory: How to
Source section: How to Resolve Login Failed [Error B]: Not Authorized Due to IdProtect Security

Alternate questions:
- How do I resolve login failed [error b]: not authorized due to idprotect security?
- How can I resolve login failed [error b]: not authorized due to idprotect security?
- Login Failed [Error B]: Not Authorized Due to IdProtect Security
- Why am I seeing Login Failed [Error B]: Not Authorized Due to IdProtect Security?

Answer:
Purpose: To assist in resolving the error "Login Failed [Error B]: Not Authorized" which occurs when a login pattern (network/IP) has been explicitly rejected under the IdProtect feature, and only an admin can override this and allow access. Video Tutorial - https://support.rediffmailpro.com/article/suspicious-login-report/ The IdProtect feature secures email access by validating login patterns (networks or IPs). If a user or admin rejects a new login pattern, the user will see: - Login Failed [Error B]: Not Authorized In such cases, only the admin can approve or restore access for the rejected pattern. - Resolution Steps: Admin to Approve Rejected Login Pattern - Admin Login Log in to the Admin Email Account with valid credentials. - Access Suspicious Login Report - Navigate to: Admin Page → Other Services → Suspicious Login Report - View Rejected Patterns Click the tab labeled “User/Admin Rejected”. In the search bar, enter the affected user’s email address and click Search. - Approve the Rejected Login Pattern Review the listed rejected login pattern(s). Click “Accept” to approve the network pattern. Once accepted, the user will be able to log in using that network.

Keywords: resolve, failed, not, authorized, idprotect, security
